# Разработка проверочного листа по безопасности данных

## Публичные данные
- Каталог недвижимости: фотографии планировок, описания квартир, жилых комплексов (площадь, этаж).
- Цены на недвижимость, условия акций.
- Публичная информация о компании: новости, пресс-релизы.
- Общая информация от УК: уведомления о собраниях, правила дома.

**Сервисы:** Онлайн-витрина.

### Риски
|Риск|Тип|Оценка|Причина|
|--|--|--|--|
|Дефейс сайта|некачественные данные|незначительный|Слабые учетные данные для CMS|
|Несанкционированное изменение цен/акций|искажение данных|значительный|Отсутствие контроля целостности контента|
|DoS/DDoS-атаки|потеря данных|критичный|Неадекватная инфраструктура для отражения атак|

## Внутренние данные
- Внутренние отчеты и метрики: эффективность менеджеров по продажам, воронка продаж.
- Внутренние коммуникации: email-переписка, документооборот.
- Архитектура IT-систем: схемы API, диаграммы взаимодействия сервисов (не содержащие учетных данных).
- История ремонтных работ в ЖК (без персональных данных).

**Сервисы:** BI-системы, CRM, tenant-core-app.

### Риски
|Риск|Тип|Оценка|Причина|
|--|--|--|--|
|Утечка к конкурентам|обесценивание данных|значительный|Отсутствие контроля доступа (любой сотрудник имеет доступ ко всему)|
|Кража интеллектуальной собственности|обесценивание данных|значительный|Ненадлежащее хранение документов|
|Внутренний саботаж или ошибка|обесценивание данных|значительный|Отсутствие DLP-систем|

## Конфиденциальные данные
- Персональные данные (PII) клиентов и собственников:
- - ФИО, паспортные данные, контакты (телефон, email).
- - Данные о сделках, правоустанавливающие документы.
- - Финансовая информация (доходы, кредитная история).
- - Платежные реквизиты.
- Данные онлайн-сделки: вся переписка, проекты договоров, статусы на разных этапах.
- Данные удаленного доступа: логины/пароли для управления "Умным домом".
- Юридические и финансовые документы: договоры с конфиденциальными условиями, судебные решения.

**Сервисы:** client-crm-app, client-mart-app, client-mart-estate-app, CRM сервиса ЖКУ.

### Риски
|Риск|Тип|Оценка|Причина|
|--|--|--|--|
|Массовая утечка ПД|утечка данных|критический|Единое хранилище данных всех систем компании|
|Мошенничество с данными клиентов|утечка данных|критический|Данные не шифруются при хранении/передаче, отсутствие MFA|
|Подмена/удаление финасовых документов|искажение данных|критический|Резервное копирование БД без шифрования|
|Несанкционированная сделка|искажение данных|критический|Резервное копирование БД без шифрования|
|Несанкционированный доступ к "Умному дому"|искажение данных|критический|Отсутствие ИБ ревью функционала|

## Секретные данные
- Ключи шифрования (TLS, database encryption).
- Учетные данные для доступа к критически важным системам (базы данных, бэкапы, панели управления ЦОД).
- Приватные ключи электронной цифровой подписи (ЭЦП), используемой для интеграции с госорганами.
- Критичные архитектурные решения (например, обходные пути в интеграциях) и пароли от систем "Умного дома".

### Риски
|Риск|Тип|Оценка|Причина|
|--|--|--|--|
|Компроментация инфраструктуры|потеря данных|критический|Ключи шифрования хранятся рядом с данными|
|Остановка бизнес-процессов|потеря данных|критический|Отсутствие ротации ключей и паролей|
|Подделка юридически значимых документов|искажение данных|критический|Приватные ключи ЭЦП не хранятся на аппаратных носителях|
